# 05. Cross Border Handle

The magic of MouseWithoutBorders is the illusion of a single contiguous desktop across physically separate machines. This is achieved by constantly polling the mouse location, mathematically detecting when it hits a bounding edge, resolving the topology matrix, and executing a control handover.

## 1. Delegated Edge Detection Logic
*(Source Reference: `App/Core/MachineStuff.cs`, `App/Core/Receiver.cs`)*

Because MouseWithoutBorders uses a **Delegated Edge Detection** architecture, the machine that actually evaluates the screen bounds is the machine whose screen the cursor is currently visibly on (the *Controlled Machine*).

Every time the mouse moves (`WM_MOUSEMOVE`), the OS global hook or the input simulator evaluates the absolute screen coordinates `(x, y)` inside `MachineStuff.MoveToMyNeighbourIfNeeded`.

### Bounding Boxes
The detection uses two primary bounds depending on settings:
- **`desktopBounds`**: The full coordinate space of all monitors on the local machine.
- **`primaryScreenBounds`**: The coordinate space of just the primary monitor.

### Trigger Threshold
The threshold for triggering a screen transition is `SKIP_PIXELS = 1`. 
The algorithm checks the boundaries:
- `x < Bounds.Left + SKIP_PIXELS` -> `MoveLeft()`
- `x >= Bounds.Right - SKIP_PIXELS` -> `MoveRight()`
- `y < Bounds.Top + SKIP_PIXELS` -> `MoveUp()`
- `y >= Bounds.Bottom - SKIP_PIXELS` -> `MoveDown()`

## 2. Matrix Resolution
*(Source Reference: `App/Core/MachineStuff.cs`)*

If an edge is crossed (e.g., `MoveRight()`), the local machine consults its *own* `LiveMachineMatrix` array (sync'd via the methods detailed in `03. state and topology management.md`).

1. It finds its *own* index in the matrix array.
2. It applies the Topology Rules:
   - **One Row (`1x4`)**: Looks at `index + 1`. If `MatrixCircle` is enabled and it is at the end of the row, it wraps around to index `0`.
   - **Two Row (`2x2`)**: Uses a hardcoded 2D grid logic. If at index `0` (top left), moving right goes to index `1` (top right). Moving down goes to index `2` (bottom left).
3. If a valid, connected machine exists in the target matrix slot, it calculates the new entry coordinate and prepares to trigger the handover.

---

## 3. Coordinate Projection
*(Source Reference: `App/Core/MachineStuff.cs`)*

Because monitors have varying resolutions, you cannot simply spawn the mouse at absolute pixel `Y=1080` on the next monitor if that monitor is only 720p. The protocol solves this using proportional projection (`ConvertToUniversalValue`).

When jumping horizontally to the Right:
1. The new `X` coordinate is clamped to the Left edge of the new machine's bounds, plus `JUMP_PIXELS = 2` (to prevent the mouse from instantly triggering a left-jump back).
2. The current `Y` pixel is converted into a normalized `0 to 65535` absolute value based on the height of the edge it just exited.
   ```csharp
   p.Y = (y - bounds.Top) * 65535 / (bounds.Bottom - bounds.Top);
   ```

---

## 4. Handover Protocol
*(Source Reference: `App/Core/Event.cs`, `App/Core/Receiver.cs`)*

Once the target machine (`newDesMachineID`) and the normalized entry coordinates are calculated, the highly synchronized handover begins:

1. **Fire NextMachine**: The Controlled Machine (whose screen border was just crossed) fires a `PackageType.NextMachine` (`121`) packet back to the Host. It embeds the new machine's ID inside the `WheelDelta` field and the `65535`-normalized entry coordinates in the `X` and `Y` fields.
2. **Host Interception**: The Host machine receives the `NextMachine` packet and executes `PrepareToSwitchToMachine()`.
3. **Hide Old Machine Cursor**: The Host sends a `PackageType.HideMouse` (`50`) packet to the previously controlled machine. This tells the old machine to make its cursor invisible.
4. **Change Control State**: The Host updates its internal `DesMachineID` state. All subsequent `Keyboard` and `Mouse` packets will now be routed with `Des = newDesMachineID`. 
5. **Notify New Machine**: The Host sends a `PackageType.MachineSwitched` (`129`) packet to the new target machine.
6. **Trigger Clipboard Sync**: If a clipboard copy occurred just before the jump, this machine switch triggers a heavy payload transfer (detailed in `06. clipboard sync.md`).
