# 09. UI Toggles & Edge Cases

The MouseWithoutBorders application contains several User Interface toggles in its Matrix settings. A common pitfall is assuming these toggles only affect superficial UI elements. In reality, they fundamentally mutate the coordinate mathematics, boundary drop-logic, and bitwise packet encodings.

This document serves as a guide for implementing parity with these toggles in a macOS client.

## 1. Move Mouse Relatively (`Setting.Values.MoveMouseRelatively`)
*(Source Reference: `App/Core/Event.cs`, `App/Core/MachineStuff.cs`)*

By default, the protocol calculates absolute cursor coordinates mapped from `0-65535` across the screen bounds.

When **Move Mouse Relatively** is enabled:
- The absolute mathematical scaling is entirely bypassed.
- Instead, the fast input thread captures the raw `dx` and `dy` movement deltas from the physical mouse hardware.
- These raw deltas are sent directly across the network inside the `MousePackage.Md.X` and `MousePackage.Md.Y` properties.
- **The Magic Constant:** To differentiate a relative coordinate from an absolute coordinate, the protocol logic requires `Math.Abs(dx) >= 100000` (which is `Event.MOVE_MOUSE_RELATIVE`). 

## 2. Block Mouse at Corners (`Setting.Values.BlockMouseAtCorners`)
*(Source Reference: `App/Core/MachineStuff.cs`)*

Because screen corners are common target areas for OS UI interactions (e.g., the Windows Start Menu or macOS Apple Menu/Hot Corners), users frequently trigger accidental machine switches.

When **Block Mouse at Corners** is enabled:
- The protocol computes a **100-pixel boundary box** (`Math.Abs(p.X - x) < 100 && Math.Abs(p.Y - y) < 100`) around the extreme coordinates of the active desktop (`WinAPI.SensitivePoints`).
- If the mouse hits the edge of the screen, but its current `Y` coordinate places it within one of these 100-pixel corner boxes, the protocol forcibly returns `Point.Empty`.
- This completely suppresses the `PackageType.NextMachine` network packet, trapping the mouse locally in the corner and preventing a handoff.

## 3. Wrap Mouse Around (`Setting.Values.WrapMouse`)
*(Source Reference: `App/Class/Setting.cs`, `App/Core/PackageType.cs`)*

The wrap-around feature allows the user to drag their mouse off the far-right edge of the final machine in the topology matrix, and have it immediately teleport to the left edge of the first machine.

When **Wrap Mouse Around** is enabled:
- The protocol modifies the topological state header.
- The `PackageType.Matrix` (`128`) base type is bitwise OR'd with the `MatrixSwapFlag` (`2`). 
- When receiving `PackageType.Matrix | PackageType.MatrixSwapFlag` (`130`), the receiver machine parses the matrix map differently, linking the final screen ID directly to the starting screen ID to form an infinite loop.

## 4. Hide Mouse at Screen Edge (`Setting.Values.HideMouse`)
*(Source Reference: `App/Core/Helper.cs`, `App/Core/Common.cs`)*

When transitioning the cursor to another machine, the local operating system must hide the physical mouse cursor so it does not appear "stuck" on the screen border.

When **Hide Mouse at Screen Edge** is enabled:
- The `HelperThread` creates a microscopic, invisible, Top-Most UI form (3x3 pixels).
- It forcefully teleports this form (and the trapped cursor) to `Y = 3` (the absolute top edge of the screen).
- This is an OS-level hack used on Windows to keep the mouse functionally hidden but still capable of receiving global hook events.

> [!TIP]
> On a macOS client, the `HideMouse` setting can simply be tied to the native `CGDisplayHideCursor()` API, provided you are managing the asynchronous `HelperThread` dispatch properly to prevent your `CGEventTap` from timing out.

## 5. Additional UI Settings & Network Toggles

The application also exposes several security and quality-of-life toggles that impact the socket lifecycle and background threads:

### Block Screen Saver on Other Machines (`Setting.Values.BlockScreenSaver`)
*(Source Reference: `App/Core/Common.cs`)*
If the user is actively working on Machine B, Machine A might hit its OS idle timeout and go to sleep. To prevent this:
- When true, Machine B stops sending standard `PackageType.Heartbeat` packets, and instead sends `PackageType.Awake` packets if it detects active `Event.RealInputEventCount` changes.
- Machine A, upon receiving `Awake`, triggers a `PokeMyself()` routine that executes micro-movements (`InputSimulation.MoveMouseRelative(x, y)`) locally. This resets the Windows idle timer without visibly moving the cursor.

### Disable Easy Mouse When Foreground Window is Fullscreen (`Setting.Values.DisableEasyMouseWhenForegroundWindowIsFullscreen`)
*(Source Reference: `App/Core/Common.cs`)*
Intended for video games or full-screen videos. When the primary display has a full-screen window active, the protocol entirely suppresses the `NextMachine` edge-transition to prevent accidental alt-tabbing or cursor loss during gameplay.

### Same Subnet Only (`Setting.Values.SameSubNetOnly`)
*(Source Reference: `App/Class/SocketStuff.cs`)*
A security toggle. During the `TcpReceiveData` socket loop, the receiver compares the incoming IP address against its local subnet mask. If they do not match, the TCP socket is immediately closed.

### Validate Remote Machine IP Address (`Setting.Values.ReverseLookup`)
*(Source Reference: `App/Class/SocketStuff.cs`)*
A secondary security toggle. Performs a Reverse DNS lookup on the incoming IP address to ensure the hostname matches the expected machine names stored in the local topology matrix before allowing the `PackageType.Handshake` ritual.

### Share Clipboard / Transfer File (`Setting.Values.ShareClipboard`, `Setting.Values.TransferFile`)
*(Source Reference: `App/Core/Clipboard.cs`, `App/Core/DragDrop.cs`)*
Hard toggles that bypass the `CheckClipboardEx` threading and the OLE Drag & Drop hooks. If `ShareClipboard` is false, `TransferFile` is automatically disabled and the out-of-band TCP socket will immediately reject connection attempts.
